01. Сравнение баз данных

| БД           | Тип            | Использование              | Плюсы                          | Минусы                         |
|--------------|----------------|---------------------------|--------------------------------|--------------------------------|
| SQLite       | Реляционная    | Мобильные и десктоп       | Лёгкая, без сервера            | Не подходит для больших нагрузок |
| MySQL        | Реляционная    | Веб-приложения            | Быстрая, популярная            | Ограничена в аналитике         |
| PostgreSQL   | Реляционная    | Сложные системы           | Надёжная, расширяемая          | Сложнее в настройке            |
| MongoDB      | NoSQL          | Big Data, JSON            | Гибкая структура               | Нет строгой схемы              |
| MS SQL Server| Реляционная    | Корпоративные системы     | Безопасность, масштабируемость | Частично платная               |

02. Разница

Текстовый документ:
- хранит данные в простом виде
- нет структуры и запросов

Сериализация:
- преобразует объекты в JSON/XML
- используется для передачи данных

Локальная БД (SQLite):
- хранит данные в таблицах
- поддерживает SQL-запросы
- обеспечивает быстрый поиск и обработку

03. Защита данных

SQLite:
- шифрование базы данных
- контроль доступа к файлу
- параметризованные запросы (защита от SQL-инъекций)

MS SQL Server:
- аутентификация пользователей
- роли и права доступа
- шифрование (TDE, SSL)
- аудит и логирование

04. Работа Python с SQLite

Общее:
- используется модуль sqlite3
- одинаковый SQL-код

Отличия:
- пути к файлам (\ в Windows, / в Linux)
- права доступа в Linux строже
- установка библиотек

Вывод:
Код одинаковый, различается только окружение.
